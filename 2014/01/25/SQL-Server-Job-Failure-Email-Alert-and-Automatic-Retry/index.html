为了忘却的纪念（一）：SQL Server 任务失败邮件报警和自动重试

想当年刚进公司的时候做的是数据仓库实习生，其中主要的职责之一便是维护日常运行的 ETL 任务，当时公司的数仓还是用的 SQL Server ＋ Hive (Hadoop) 的混合平台，其中 SQL Server 中跑了大量的数据消费层的任务，所以写了一些小的运维脚本工具。

之所以题目中使用了为了忘却的纪念，是因为最后大家终于忍不了微软套件的封闭，于是下决心把数据部门中所有的 Windows 服务器都干掉，全面拥抱开源。SQL Server 见证了我职业生涯最初期的表现，所以我有必要写两篇文章纪念一下。

Update @ 2016-05-13：微软先收购了 Revolution，又推出了 SQL Server on Linux，然后又在 VS 中集成了基于 R 语言的工具，不管是出于什么原因，大巨头越来越开放也是一件值得高兴的事呢。

后来发现，有些失败的任务，过一段时间之后简单地重新运行一下即可，于是乎又写了一个自动重试的 SP，也是每个小时的第 10 分钟运行，逻辑是对那些可以重新运行解决问题的任务，自动重试。
