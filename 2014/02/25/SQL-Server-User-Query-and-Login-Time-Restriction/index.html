为了忘却的纪念（二）：SQL Server 限制特定时间段内用户的查询和登录

有一天早上醒来，发现大部分的任务都在等待依赖阶段，比往常晚了五六个小时。查原因发现原来是 BA 某同学的凌晨提交的一个查询过大，将非常重要的一张表锁住了，导致新一天的数据无法插入进来。因为凌晨 0-6 点这段时间基本上是系统任务的运行时间段，所以为了解决这个问题，当时决定这段时间暂停向特定用户提供数据查询服务，以保证系统数据流的正常运行。

限制特定用户的登录

清理特定用户已提交的查询

有的时候可能用户已经在零点之前提交了查询，如果查询较大的话同样会影响系统任务的运行，所以在限制登录的同时还要把这部分已经提交的查询杀掉。这一步使用的是 SP，每天零点运行一次即可。
