OpenTSDB Digest：HBase 数据的读取

了解了 OpenTSDB HBase 数据表的设计和数据写入的过程，数据读取的过程就比较简单了，还是上代码。

1. tsdb-uid 表的查询

OpenTSDB 自身提供了一个数据查询的 UI，在输入 metric / tagk / tagv 的前缀的时候，会有数据库中已存数据的提示，这个过程中 OpenTSDB 实际上对 HBase 中的 tsdb-uid 表进行了查询，过程就是一次设定 start end row key 的扫描。

class: net.opentsdb.core.TsdbQuery

2. tsdb 表的查询

2.1 设定 scan 的 start & end key

OpenTSDB 的主要查询是对 tsdb 表的查询，首先拿到查询条件中的 metric 和 timestamp 信息，OpenTSDB 会先设定好 scan 的 start & end key。

为了保证 downsample 等操作的正常进行，OpenTSDB 会根据给定的 timestamp 向前向后各多取一行（一个小时）的数据，详见 getScanStartTimeSeconds() 和 getScanEndTimeSeconds() 两个方法。

class: net.opentsdb.core.TsdbQuery

2.2 设定 scan 的正则过滤

除了 metric 和 timestamp 这两个条件，tag 的过滤是通过正则过滤来实现的。假设我们的查询条件中有两个 tag：{ 0 0 1 0 0 2 } 和 { 4 5 6 9 8 7 }，最后生成的正则应该是：^.{7}(?:.{6})*\\Q\000\000\001\000\000\002\\E(?:.{6})*\\Q\004\005\006\011\010\007\\E(?:.{6})*$。

class: net.opentsdb.core.TsdbQuery
